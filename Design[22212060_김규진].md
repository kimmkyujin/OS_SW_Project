## 1. Introduction

본 문서는 Analysis 단계 문서에 이어지는 든든(Deun Deun) 어플리케이션의 Design 단계 문서이다.

### 1.1 Summary
든든(Deun Deun)은 자취생과 1인 가구를 위한 맞춤형 요리 보조 어플리케이션으로, 사용자의 보유 재료 및 조리도구 상황에 맞게 레시피를 동적으로 변환하여 제공하는 것을 핵심 기능으로 한다. Analysis 단계에서 정의된 10개의 Use Case를 바탕으로, 본 Design 문서에서는 시스템을 구성하는 클래스 구조, 주요 기능 흐름(Sequence Diagram), 상태 전이(State Machine Diagram)를 구체적으로 정의한다.


### 1.2 Important Points of Design

- **레시피 변환 중심 설계:** 사용자의 인원수·재료·조리도구 선택에 따라 레시피를 동적으로 변환하는 것이 든든 앱의 핵심 기능이며, 모든 다이어그램에서 이 흐름을 중심으로 설계한다.

- **권한 기반 접근 제어:** 일반 사용자(USER)와 관리자(ADMIN)의 역할을 명확히 구분하여, 각 기능에 대한 접근 범위를 제한한다.

- **데이터 무결성 유지:** 클래스 간 다대다 관계를 연관 클래스로 분리하고, 원본 데이터를 직접 수정하지 않는 방향으로 설계한다.

- **예외 처리 흐름 명확화:** 네트워크 오류, 빈 입력, 중복 데이터 등 Use Case에서 정의된 예외 시나리오를 각 다이어그램에서 구체적으로 표현한다.

### 2.1 User

| Attributes | details|
|---|---|
| userId: int | 사용자 고유 식별자 |
| loginId: String | 로그인 시 사용하는 아이디 |

### 2.6 RecipeIngredient

| Attributes | details|
|---|---|
| recipeId: int | 레시피 식별자 |
| ingredientId: int | 식재료 식별자 |
| amount: float | 해당 레시피에서 사용하는 재료의 양 |
| unit: String | 해당 레시피에서 사용하는 재료의 단위 |

| Description |
|---|
| RecipeIngredient 클래스는 Recipe와 Ingredient 간의 다대다 관계를 해소하는 연관 클래스이다. 같은 재료라도 레시피마다 필요한 양(amount)과 단위(unit)가 다르기 때문에, 해당 정보를 Recipe도 Ingredient도 아닌 이 연관 클래스에서 관리한다. |

### 2.8 RecipeAdapter

| Attributes |details |
|---|---|
| baseRecipe: Recipe | 변환의 기반이 되는 원본 레시피 객체 |
| targetServings: int | 사용자가 선택한 목표 인원수 |
| selectedIngredients: List\<Ingredient\> | 사용자가 선택한 대체 식재료 목록 |
| selectedTools: List\<CookingTool\> | 사용자가 선택한 대체 조리도구 목록 |

| Methods | |
|---|---|
| adaptServings(servings: int): void | 목표 인원수에 맞게 재료 계량 수치를 비례 계산하여 업데이트 |
| adaptIngredients(ingredients: List\<Ingredient\>): void | 선택된 대체 재료로 재료 목록을 교체 |
| adaptTools(tools: List\<CookingTool\>): void | 선택된 대체 도구로 조리도구 목록을 교체 |
| generateAdaptedRecipe(): Recipe | 변환된 인원수·재료·도구를 반영한 새 Recipe 객체를 생성하여 반환 |

| Description |
|---|
| RecipeAdapter 클래스는 사용자가 선택한 인원수, 재료, 조리도구 옵션에 따라 기존 레시피를 동적으로 변환하여 맞춤형 레시피를 생성하는 핵심 클래스이다(Use Case #5). |

### 2.10 Inquiry

| Attributes | |
|---|---|
| inquiryId: int | 문의 고유 식별자 |
| userId: int | 문의를 작성한 사용자의 식별자 |
| category: String | 문의 카테고리 (레시피 추천 / 버그 제보 / 기타) |
| title: String | 문의 제목 |
| content: String | 문의 상세 내용 |
| status: String | 처리 상태 (대기 / 완료) |
| createdAt: DateTime | 문의 작성 일시 |
| answer: String | 관리자가 작성한 답변 내용 |
| answeredAt: DateTime | 관리자 답변 등록 일시 |

| Methods | |
|---|---|
| submit(): void | 작성된 문의 내용을 DB에 전송 및 저장 |
| updateStatus(status: String): void | 문의 처리 상태를 변경. 관리자 전용 기능 |
| addAnswer(answer: String): void | 관리자의 답변 내용을 저장하고 answeredAt을 현재 시각으로 설정 |

| Description |
|---|
| Inquiry 클래스는 사용자가 서비스 이용 중 제출하는 버그 제보, 레시피 요청 등의 문의사항을 저장하는 클래스이다. 사용자의 문의 제출(Use Case #9)과 관리자의 답변 처리(Use Case #10)를 하나의 클래스에서 관리하며, status 속성으로 처리 상태(대기/완료)를 구분한다. |

### 2.11 Recommendation

| Attributes | |
|---|---|
| period: String | 추천 기간 구분 (주간 / 월간 / 연간) |

| Methods | |
|---|---|
| getTopRecipes(period: String, limit: int): List\<Recipe\> | 지정 기간 동안 점수가 높은 상위 limit개의 Recipe 목록을 반환 |

|---|
