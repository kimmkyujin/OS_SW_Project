## 1. Introduction
본 문서는 자취생 요리 보조 어플리케이션인 든든의 분석결과에 대한 내용을 담고있다. <br/>
기존의 레시피 어플들이 표준적인 조리환경만을 가정하기에 이를 바꿔보고자, 사용자의 현재 상황에 맞춘 유동적인 레시피 변환 어플을 설계하였다. 
1) Prominent Features
- 해당 어플은 단순하게 레시피를 나열하는 것이 아닌 사용자가 현재 가진 도구와 재료 상황에 따라 레시피를 변경하여 최적의 레시피를 제공한다. 또한 요리초보들이 겪는 재료낭비와 요리실패의 상황을 줄여주며, 1인 가구의 건강한 식생활 자립을 돕는 가치를 지닌다.
2) Goals
- Analysis를 통해 이뤄야할것은 먼저 Actor들과 Sever 그리고 System이 어떠한 상호작용으로 이루어지는지 그 시나리오를 명확히 할것이다. System이 User의 요청에 따라 옳바른 답을 내놓도록하는 로직을

#### Use case #1: Login

**GENERAL CHARACTERISTICS**

| **Field** | **Description** |
| :---: | --- |
| **Summary** |사용자가 아이디와 비밀번호를 입력해 시스템 접속 권한을 얻는 기능|
| **Scope** | Deun Deun |
| **Level** | User level|
| **Status** | Analysis |
| **Primary Actor** | User, Administrator|
| **Preconditions** | <br/> 앱을 실행하여야 한다. <br/> 사용자가 회원가입이 되어있어야 한다. <br/>인터넷에 연결되어 있어야 한다. |
| **Trigger** |앱을 실행시킬떄|
| **Success Post Condition** |해당 로그인 정보에 맞는 권한을 부여받고 메인화면으로 들어가게 되며, 앱을 사용할 수 있게된다.|
| **Failed Post Condition** | 권한을 부여받지 못하며, 로그인 화면이 그대로 유지된다.|

**MAIN SUCCESS SCENARIO**

| **Step** | **Action** |
| :---: | --- |
| 1 |사용자가 로그인화면에서 아이디와 비밀번호를 입력하고 로그인 버튼을 누른다.|
| 2 | 입력한 정보가 서버의 데이터베이스의 회원정보에 존재하는지 확인한다.|
| 3 | 정보가 존재하면 로그인 성공 처리후 메인화면을 띄운다. (Use case 종료)|

**EXTENSION SCENARIOS**

| **Step** | **Branching Action** |
| :---: | --- |

|2|2a. 입력한 정보가 회원정보와 일치하지 않는 경우 <br/>. . . 2a.1. 화면에 "아이디 또는 비밀번호가 틀렸습니다"라는 메세지를 띄운다.|

#### Use case #2: Register

**GENERAL CHARACTERISTICS**

| **Field** | **Description** |
| :---: | --- |
| **Summary** | 신규 사용자가 시스템을 이용하기 위해 계정 정보를 등록하는 기능|
| **Scope** | Deun Deun |
| **Level** | User level|
| **Status** | Analysis|
| **Primary Actor** | User|
| **Preconditions** | 앱이 실행되어야 한다. <br/> 인터넷이 연결되어 있어야 한다.|
| **Trigger** | 로그인 화면에서 회원가입 버튼을 눌렀을 때|
| **Success Post Condition** | 입력한 회원정보가 서버의 데이터베이스에 저장되고 회원가입이 완료된다.|
| **Failed Post Condition** | 회원가입에 실패하기에 입력한 정보가 저장되지 않고 회원가입 화면이 그대로 유지된다. |

| 2 | 회원가입에 필요한 정보 입력 후 가입 버튼을 누른다.|
| 3 | 시스템이 입력한 아이디가 서버의 데이터베이스에 이미 존재하는지 중복검사를 수행한다.|
| 4 | 중복이 아니면 회원정보를 데이터베이스에 저장 후 로그인 화면을 띄운다. (Use case 종료) |

**EXTENSION SCENARIOS**

| 3 | 3a. 입력한 아이디가 데이터베이스에 이미 존재하는 경우 <br/> . . . 3a.1. 화면에 "이미 사용 중인 아이디입니다"라는 메세지를 띄운다. |

#### Use case #3: Recipe Search

**GENERAL CHARACTERISTICS**

| **Field** | **Description** |
| :---: | --- |
| **Summary** | 사용자가 원하는 요리명이나 식재료를 검색하여 레시피 목록 및 상세 정보를 조회하는 기능|
| **Scope** | Deun Deun |
| **Level** | User level|
| **Status** | Analysis|
| **Primary Actor** | User|
| **Preconditions** | 사용자가 시스템에 로그인되어 있어야 한다.|
| **Trigger** | 메인 화면의 검색창에 키워드를 입력하고 검색 버튼을 눌렀을 때|
| **Success Post Condition** | 검색어와 일치하는 레시피 목록, 상세 정보가 화면에 출력된다.|
| **Failed Post Condition** | 화면에 아무 레시피도 나오지 않으며 중앙에 "해당 레시피가 없습니다" 라는 메세지가 나온다. |

**MAIN SUCCESS SCENARIO**

| **Step** | **Action** |
| :---: | --- |
| 1 | 사용자가 검색창에 요리 이름이나 식재료 키워드를 입력하고 검색 버튼을 누른다. |
| 2 | 시스템은 서버의 데이터베이스에서 입력된 키워드와 일치하는 레시피 정보를 조회한다. |
| 3 | 조회된 레시피 목록을 화면에 출력한다. |
| 4 | 사용자가 특정 레시피를 선택하면 상세 조리법 화면으로 이동한다. (Use case 종료) |

**EXTENSION SCENARIOS**

| **Step** | **Branching Action** |
| :---: | --- |

| 2 | 2a. 데이터베이스에 검색어와 일치하는 레시피가 없는 경우 <br/> . . . 2a.1. 화면중앙에 "해당 레시피가 없습니다"라는 글이 적혀있으며 목록에는 아무 레시피도 나오지 않는다. |

| **Success Post Condition** | 변경할 수 있는 옵션들의 목록이 뜬다|
| **Failed Post Condition** | 기존 레시피 화면이 그대로 유지된다. |

| **Step** | **Action** |

| 2 | 시스템은 사용자가 '1'에서 누른 버튼에 따라 해당 레시피에 적용 가능한 대체 옵션을 화면에 띄운다.|
| 3 | 사용자가 각 항목에 대해 원하는 옵션을 선택한 후 적용 버튼을 누른다.|

| **Scope** | Deun Deun (든든) |
| **Level** | User level |

| **Failed Post Condition** | 변경한 옵션이 없거나 오류 발생 시, 기존 레시피 정보를 유지하거나 안내 메시지를 출력함 |

| :---: | --- |
| 1 | 사용자가 원하는 옵션을 모두 택하고 '적용' 버튼을 누른다. |
| 2 | 호출된 데이터에 맞추어 화면상의 재료 수량, 조리 순서, 이미지 등의 정보가 실시간으로 업데이트된다. |
