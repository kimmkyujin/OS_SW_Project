## 1. Introduction
본 문서는 자취생 요리 보조 어플리케이션인 든든의 분석결과에 대한 내용을 담고있다. <br/>
기존의 레시피 어플들이 표준적인 조리환경만을 가정하기에 이를 바꿔보고자, 사용자의 현재 상황에 맞춘 유동적인 레시피 변환 어플을 설계하였다. 
1) Prominent Features
- 해당 어플은 단순하게 레시피를 나열하는 것이 아닌 사용자가 현재 가진 도구와 재료 상황에 따라 레시피를 변경하여 최적의 레시피를 제공한다. 또한 요리초보들이 겪는 재료낭비와 요리실패의 상황을 줄여주며, 1인 가구의 건강한 식생활 자립을 돕는 가치를 지닌다.
2) Goals
- Analysis를 통해 이뤄야할것은 먼저 Actor들과 Sever 그리고 System이 어떠한 상호작용으로 이루어지는지 그 시나리오를 명확히 할것이다. System이 User의 요청에 따라 옳바른 답을 내놓도록하는 로직을

| **Success Post Condition** | 선택한 레시피의 상세 정보(이미지, 재료, 조리법 등)가 화면에 표시되며, 사용자는 스크롤을 통해 전체 내용을 읽을 수 있음.  |

| :---: | --- |

| 2 | 시스템은 해당 레시피의 전체 데이터(기본 조리법, 인원수, 재료 목록 등)를 화면에 출력한다. |
| 3 | 화면에는 스크랩(use case #8)을 위한 별버튼과 레시피 변경(use case #5)을 위한 옵션버튼이 있다.
| 4 | 사용자는 화면을 스크롤하며 레시피의 상세 내용을 열람한다. |
| 5 | 사용자가 상세 페이지를 나가면 use case는 종료된다.|

**GENERAL CHARACTERISTICS**

| **Summary** | 메인 화면의 추천 버튼을 통해 기간별(주간/월간/연간) 인기 레시피 TOP 5 목록을 제공하는 기능 |

| **Success Post Condition** | 사용자가 선택한 항목(이번 주/이번 달/올해)에 맞는 인기 레시피 TOP 5가 화면에 표시됨 |

**MAIN SUCCESS SCENARIO**

## 3. Domain Analysis

**1) User**
사용자 클래스. 회원가입 및 로그인 정보를 담고 있으며, 일반 사용자와 관리자 권한을 구분하여 시스템 접근 범위를 결정하는 클래스다.

**2) Recipe**
레시피 클래스. 요리명, 기본 인분 수, 난이도, 상세 조리 순서 등 요리의 핵심 데이터를 포함하며 시스템의 가장 중심이 되는 정보를 관리하는 클래스다.

**3) Ingredient**
식재료 클래스. 레시피에 필요한 재료 정보를 담고 있다. 각 재료의 중요도(필수/선택), 표준 단위(g, 큰술 등) 및 신선도 유지를 위한 보관법 정보를 포함한다.

**4) CookingTool**
요리도구 클래스. 조리에 필요한 도구 정보를 관리한다. 해당 도구가 없을 때 사용할 수 있는 대체 도구 정보와 조리 후 도구별 올바른 세척 및 관리법을 포함한다.

**5) RecipeAdapter**
레시피 변환 클래스. 사용자가 선택한 인원수, 보유 재료, 조리 도구 옵션에 따라 기존 레시피의 수치 및 조리법을 변경하여 맞춤형 레시피를 생성하는 기능을 담당한다.

**6) Scrap**
스크랩 클래스. 사용자가 나중에 다시 보고 싶은 레시피를 저장하는 기능을 수행한다. 어떤 사용자가 어떤 레시피를 저장했는지에 대한 정보를 관리한다.

**7) Inquiry**
문의사항 클래스. 서비스 이용 중 발생하는 버그 제보나 레시피 요청 내역을 저장한다 문의유형, 내용 및 관리자의 답변 상태와 답변 내용을 포함한다.

**8) Recommendation**
추천 클래스. 시스템 내의 조회수와 스크랩 수 데이터를 분석하여 주간, 월간, 연간 단위로 사용자에게 인기 있는 레시피 TOP 5 목록을 산출하는 로직을 관리한다.
